Токен в формате "Bearer {{TOKEN}}"

Авторизация пользователей
Авторизация пользователя производится с использованием специального токена, который налогоплательщик может сгенерировать в <<Кабинете налогоплательщика>>:
   1. Перейти в раздел <<Генерация токена>> (рис. 1)
   2. Нажать на кнопку <<Сгенерировать токен>> (рис. 1)
   3. Заполнить данные для генерации токена (рис. 2)
         a. Срок действия токена
         b. Выбрать агрегатора, через которого будет использован токен
         c. Выбрать системы, в которых будет использован токен
         d. Сохранить
   4. Скачать токен (рис. 1)
Сохраненный токен можно использовать в информационной системе агрегатора для авторизации.

Описание ошибок
1. Accept Or Reject
URL: /api/command/invoice/accept-or-reject

1.1 
HTTP status: 400 Bad Request
Response Message:
{
    "message": "Validation error",
    "errors": {
        "statusCode": "Incorrect status code"
    }
}
Возникает, когда в запросе отправлен некорректный statusCode, допустимые значения statusCode для accept-or-reject
   * 40 (принят)
   * 50 (отклонен)

1.2
HTTP status: 404 Not Found
Response Message:
{
    "message": "Документ с кодом статуса 30 и UUID %s не обнаружен"
}
Возникает, когда в массиве documentUuids был передан недействительный UUID документа

1.3
HTTP status: Bad Request
Response Message:
{
    "message": " Документ с UUID %UUID% уже в статусе %status name%"
}
Возникает, когда в statusCode передано значение, которое уже присвоено данному документу

1.4
HTTP status: 400 Bad Request
Response Message:
{
    "message": " Статус документа с UUID %s не может быть изменен, так как его статус не равен 30 "
}
Возникает, когда документ, статус которого клиент пытается изменить не является Отправлен (30)
